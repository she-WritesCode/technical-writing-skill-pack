# Document Templates

Use these structures to keep the draft shape predictable.

## API overview

1. What the API does
2. Who it is for
3. Core concepts or objects
4. Authentication and access
5. Common workflow
6. Key limits or caveats
7. Links to detailed reference

## Endpoint or operation doc

1. Purpose
2. Method and path
3. Authentication
4. Request parameters
5. Request body
6. Response body
7. Errors
8. Example request
9. Example response
10. Notes and caveats

## Release note

1. What changed
2. Why it matters
3. Affected endpoints or objects
4. Backward compatibility impact
5. Required user action
6. Links to updated reference

## Migration note

1. Old behavior
2. New behavior
3. Breaking changes
4. Required changes for integrators
5. Timeline or deprecation status
6. Verification steps

## Quick reference guide

1. Top-level objects
2. Important fields
3. Relationships between objects
4. Short meaning notes
5. Links to canonical docs
