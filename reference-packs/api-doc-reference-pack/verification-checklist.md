# Verification Checklist

Use this before calling a draft final.

## Factual checks

- Are all major claims supported by a trusted source or reviewer confirmation?
- Are names of endpoints, fields, objects, and parameters current?
- Are auth, permissions, quotas, and pagination rules correct?
- Are deprecation or migration statements verified?

## Coverage checks

- Does the draft omit any required prerequisite or caveat?
- Does it explain the task the user is actually trying to do?
- Does it link to deeper reference where needed?

## Example checks

- Does each example reflect supported behavior?
- Do request and response examples match the described schema?
- Is there at least one failure or edge-case example where useful?

## Consistency checks

- Is terminology consistent with the rest of the doc set?
- Are code terms linked or formatted consistently?
- Are headings shaped consistently with similar docs?

## Human review packet

Send the reviewer:

- the draft
- the source inventory
- the uncertainty register
- 3 to 5 specific review questions

## Status labels

- `draft-internal`
- `ready-for-review`
- `reviewed-needs-revision`
- `verified-final`
