# Publishing Checklist

Use this checklist before pushing the repo to GitHub.

## Repository metadata

- Pick the final repository name.
- Confirm the license is the one you want to publish under.
- Add a short GitHub description.
- Add topics such as `ai-agent`, `documentation`, `technical-writing`, `api-docs`, and `prompt-engineering`.

## Content review

- Confirm `SKILL.md` matches the workflow you want others to use.
- Confirm the reference pack files are complete and internally consistent.
- Confirm `skills.sh.json` matches the public title, description, tagline, and topics you want shown.
- Remove any local-only notes or scratch files.
- Check that example prompts, placeholders, and templates are intentional.

## Installation clarity

- Verify the install path in `README.md`.
- Confirm the usage examples still match the skill behavior.
- Add a screenshot or repo social card later if you want a better landing page on GitHub.

## Release readiness

- Create the GitHub repository.
- Push the folder contents.
- Tag the first release if you want a stable install target.
- Optionally add a changelog once the skill starts evolving.
